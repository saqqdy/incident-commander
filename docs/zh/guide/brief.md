# 事故简报

事故简报是一种简洁的、面向利益相关者的格式，总结故障的关键事实。

## 使用方式

```text
/incident brief
```

## 输出示例

```text
🔔 事故简报
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  标题:    user-service API 破坏性变更
  严重等级: SEV2
  持续时间: 35 分钟
  影响:   约 5000 用户受影响
  状态:   ✅ 已解决
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  发生了什么:
    一次部署 (v2.5.0) 移除了 /api/users 端点，
    导致下游消费者出现 500 错误。

  处理措施:
    于 10:30 UTC 回滚生产环境至 v2.4.0。
    错误于 10:35 UTC 恢复。

  后续步骤:
    - 添加消费者契约测试 (P1)
    - 实施金丝雀部署 (P1)
```
