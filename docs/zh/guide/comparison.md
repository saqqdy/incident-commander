# 方案对比

Incident Commander 与现有工具和流程有何不同？

## vs PagerDuty / Opsgenie

| 维度 | PagerDuty / Opsgenie | Incident Commander |
|------|---------------------|-------------------|
| 故障检测 | ✅ 告警聚合 + 值班调度 | ❌ 不是告警工具（专注于分析） |
| 时间线构建 | ⚠️ 仅聚合告警，无代码变更 | ✅ 融合提交、部署、错误、指标、日志 |
| 根因分析 | ❌ 无推理 — 依赖人工 | ✅ AI 交叉验证因果链 + 置信度标签 |
| Post-Mortem | ⚠️ 提供空白模板 | ✅ 自动生成已填充的报告 — 审查即可发出 |
| 部署方式 | SaaS，付费 | 本地，开源，免费 |
| **定位** | 告警接入 + 值班编排 | 故障分析 + 复盘文档 |

> Incident Commander 补充而非替换 PagerDuty — PagerDuty 检测问题，Incident Commander 分析问题并生成复盘。

## vs Datadog Incident Management

| 维度 | Datadog IM | Incident Commander |
|------|-----------|-------------------|
| 数据源 | ✅ 深度绑定 Datadog 生态 | ✅ 开放集成：GitHub / Sentry / Grafana / Kibana |
| AI 分析 | ⚠️ 有限的异常检测 | ✅ 深度因果推理 + 替代假设 |
| 跨平台关联 | ❌ 仅限 Datadog 数据 | ✅ 跨平台交叉验证 |
| 成本 | 企业级定价 | 开源，免费 |
| **定位** | Datadog 内的故障管理 | 适用于任何技术栈的故障分析 |

## vs 手动复盘流程

| 维度 | 手动流程 | Incident Commander |
|------|---------|-------------------|
| 时间线构建 | 30–60 分钟，在工具间切换 | 10 秒，自动生成 |
| RCA 深度 | 取决于个人经验 | 系统化 AI 推理 + 自动建议替代方案 |
| Post-Mortem 撰写 | 从零开始 1–2 小时 | 10 分钟审查和编辑 |
| 知识保存 | 散落在人和系统中 | 结构化、可搜索、可复用的归档 |
| 一致性 | 因作者而异 | 标准化模板 + 统一的输出格式 |

## 核心差异

1. **AI 语义推理 vs 规则匹配** — 只有 AI 能理解"这段代码的行为变了"，跨越千变万化的日志和错误
2. **开放生态 vs 供应商锁定** — 自由组合 GitHub + Sentry + Grafana + Kibana — 无单一平台锁定
3. **Claude Code 原生 vs 独立工具** — 直接在你的开发工作流中使用 — 无需切换上下文
