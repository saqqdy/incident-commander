# 优雅降级

Incident Commander 在数据源不可用时优雅降级，而非崩溃或产生空结果。

## 降级链

```text
采集器降级：
  MCP Server ──(不可用)──▶ CLI 工具 ──(不可用)──▶ 引导手动输入 ──(跳过)──▶ 标记数据缺失

整体降级：
  全自动模式 ──(部分源不可用)──▶ 半自动模式（可用源采集 + 手动补充）
              ──(全源不可用)──▶ 纯交互模式（问答采集信息）
```

## 置信度调整

RCA 置信度根据可用数据自动调整：

| 可用数据源 | 最高置信度 | 原因 |
|-----------|-----------|------|
| 3+ 个数据源 | 🟢 高 | 可进行交叉验证 |
| 2 个数据源 | 🟢 高 | 部分交叉验证 |
| 1 个数据源 | 🟡 中 | 无法交叉验证 |
| 仅手动提供 | 🔴 低 | 基于用户叙述 |

## 最佳实践

1. **至少配置 2 个数据源** — 交叉验证显著提升 RCA 质量
2. **保持 gh CLI 认证** — 它是最可靠的降级方案
3. **查看缺口标记** — Post-Mortem 中的 `[MISSING]` 部分需要手动补充
4. **定期运行 `/incident config`** — 确保数据源保持连接

详细说明请参考 [英文文档](/advanced/degradation)。
